## Hypothesis

Establish baseline val_bpb with the simplified vanilla GPT model on Kaggle P100. This is the first run after stripping train.py to basic transformer (no Muon, no value embeddings, no sliding window, no softcap). Using AdamW with GradScaler for P100 fp16 compatibility.

## Changes

Initial simplified train.py with:
- Vanilla GPT: Embedding + 6x (RMSNorm + MHA w/ RoPE + RMSNorm + MLP w/ GELU) + RMSNorm + Linear head
- Standard AdamW optimizer (LR=3e-4, weight_decay=0.1, betas=(0.9, 0.999))
- GradScaler for fp16 training, gradient clipping (max_norm=1.0)
- SDPA fallback (P100 doesn't support FA3)
- DEPTH=6, DEVICE_BATCH_SIZE=16 (fit 16GB P100 VRAM)

## Result

Baseline established at val_bpb=2.7992 on P100. Training completed successfully with 33 steps in 5 minutes. Loss decreased from ~9.01 (random init, ln(8192)) to ~8.19 by end of training. The model is very undertrained (only 17.3M tokens seen) due to P100's slow throughput (~38K tok/sec).

## Notes

- P100 requires torch 2.4.1+cu118 (modern PyTorch drops sm_60 support)
- fp16 autocast + GradScaler needed (P100 doesn't support bf16)
- torch.compile disabled (Triton requires sm_70+)
- Only 33 training steps in 5 min budget — each step takes ~14s with 16 grad_accum steps
- VRAM usage is only 6.4GB out of 16GB — room to increase DEVICE_BATCH_SIZE or DEPTH
- WandB logging confirmed working: https://wandb.ai/kuncevich-andrew-work-andrewk/autoresearch
- Key bottleneck: low throughput means very few optimization steps per run
